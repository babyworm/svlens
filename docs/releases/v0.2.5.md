# svlens v0.2.5 release notes

`svlens` `v0.2.5` is the patch release that stabilizes large-design adoption
after the initial trust / install / help rollout.

## Highlights

### 1. High-confidence false alarms reduced on large SoCs

The main false-alarm reductions validated against large open SoC designs are:

- optional `exists_*` connectivity inputs suppressed only when disabled by
  an exact `exists_req_i == 1'b0` companion tie-off
- intentional `isochronous_4phase_handshake` CDC crossings downgraded from
  blocking severity
- intentional `syncreg` CDC crossings downgraded from blocking severity
- combinational-only debug/JTAG mux logic no longer misclassified as FF state

Observed replay results on the tuned configurations:

| Design | Metric | Before | After |
|---|---|---:|---:|
| Cheshire | conn `ERROR` | 8 | 0 |
| Cheshire | cdc `VIOLATION` | 2 | 0 |
| PULPissimo | cdc `VIOLATION` | 4 | 0 |

### 2. Test harness hardening for repeated validation

- `compileSV(...)` temporary source files now use unique names
- shell integration tests now use `mktemp -d` output roots
- repeated and concurrent validation runs no longer share fixed `/tmp` paths

### 3. GitHub Actions runtime modernization

- `actions/checkout` updated to `v5`
- `actions/cache` updated to `v5`
- workflows are aligned with the GitHub-hosted Node 24 transition guidance

### 4. Large-SoC rollout guidance

New documentation:

- [`docs/waiver-baselines.md`](../waiver-baselines.md)

This adds:

- connectivity baseline guidance
- CDC waiver strategy guidance
- recommended CI rollout policy for large SoCs
- Cheshire / PULPissimo replay interpretation
