# svlens v0.2.8 release notes

## Highlights

### 1. CDC detection quality — Track A landed (Round 7-26)
20-round hardening campaign that closed 18+ false-negative classes
across the CDC analyzer. Now ships with 52 golden fixtures (paired
pos/neg pairs) and 12 OSS RTL modules validated externally
(ZipCPU/OpenTitan/CVA6/BlackParrot).

Specific gaps closed:

* **AST utilities** -- `splitAssignmentByLHS` recursive helper handles
  5+ levels of nested concat-LHS, parameter / TypeParameter / EnumValue
  references are filtered out of fanin, hierarchical references return
  full dotted paths.
* **FF classifier** -- O(D) ancestor walk over `clock_db.net_by_path`
  (was O(D*N)); `fanin_populated` invariant distinguishes stub vs.
  walked-but-empty FFNodes.
* **Clock tree** -- `isPortUsedAsClock` restricts port-as-clock
  detection to `always_ff` event control with explicit edge; reset
  port filter; scope-qualified de-dup key (parent_scope + signal)
  prevents shadowed `clk` names from over-merging.
* **Connectivity** -- hierarchical-reference cross-domain reads,
  generate-array dotted path lookup with dual-variant normalization,
  parent-prefix + idx + leaf-suffix scan with dot-count guard.
* **Sync verifier** -- bounded BFS forward extension (MAX_DEPTH=6,
  Cummings SNUG 2008 citation), `isSingleSourceFF` predicate handles
  reset-only FFs and multi-fanin merges, comb-guard with same-domain
  local enable stays TwoFF (not PulseSync).

### 2. CDC: --emit-sva CLI flag
New informational SystemVerilog Assertion emitter:

* VIOLATION crossings with `SyncType::None` produce a `cover property`
  watching `!$stable(<source>)` against the dest clock. Cover (not
  assert) so simulation does not false-fail on legitimate async
  toggling; the cover hits when the toggle event is exercised.
* Verified synchronizers (TwoFF/ThreeFF/PulseSync etc.) produce a
  documentation-only block — structure is statically verified, so
  no runtime property is needed.
* Identifier sanitizer collapses non-alphanumeric / non-underscore
  characters to `_` so report ids like `VIOLATION-1` become valid
  SVA property names (`VIOLATION_1`).

Use: `svlens cdc --top <module> rtl/*.sv --emit-sva cdc_assertions.sva`

### 3. CDC: clock_groups auto-waive (single-group implicit other)
`set_clock_groups -physically_exclusive` and `-logically_exclusive`
now auto-waive crossings (`-asynchronous` still requires sync stages
by design). Single-group declaration triggers an implicit "every
other clock" waiver against the named group.

### 4. conn / metrics: paired-fixture methodology applied
First-pass fixtures codifying gaps for follow-up hardening:

* `conn_modport_width_pos.sv` / `_neg.sv` -- 8-bit modport signal
  truncation into a 16-bit consumer port through a hierarchy boundary.
  Currently exit=0 (codified gap).
* `metrics/lhs_concat_for_loop.sv` -- baseline for the for-loop +
  LHS-concat interaction; per-root sum 14, max depth >= 3.

### 5. Documentation
* `docs/design/feature-roadmap.md` Track A landed-items table with
  10 sub-entries mapping each closed gap to the round and fixture.
* `docs/schema/cdc_report.md` -- 3 previously undocumented fields
  (`relationship`, `rationale`, `timing_basis_ns`) and 9 sync_type
  enum values.
* `tests/cdc/README.md` -- external coverage table (12 OSS modules
  with FF count, V/C/INFO triple, sync_type) + per-fixture
  performance baseline.

### 6. Infrastructure
- 541 ctest entries (was 504 in 0.2.7), all PASS deterministically.
- 526 unit cases / 1761 assertions across all modes.
- 0 warnings / 0 errors across svlens-{cdc,conn,metrics}-lib.
