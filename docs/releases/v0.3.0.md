# svlens v0.3.0 release notes

Minor version bump aggregating Round 28-32 review-driven hardening
plus the substantive modport-width inference fix (US-R05). All real
issues from two independent code-reviewer passes are closed.

## Highlights

### 1. CDC: --emit-sva CLI flag, hardened
The Round 27 SVA emitter is now production-ready after three
review-driven fixes:

* **Leading-dot expression sanitizer**: `svaExpressionSafe(s, out)`
  strips leading dots from source signal paths and rejects paths
  with operator-class characters. Cover properties using the
  cleaned expression are emitted only when the body is valid SV;
  otherwise a comment-only block is written.
* **Failure surfacing**: `generateSVA` now returns
  `[[nodiscard]] bool`. `emitCdcReports` consults the return value
  and writes a stderr warning on file-open failure (parent dir
  missing, permission denied). Analysis exit code is unaffected;
  file-write failure is a warning, not an analysis failure.
* **Identifier hardening**: `svaSanitize` prefixes `_` when the
  result begins with a digit (SV identifiers cannot start with
  digits). Property-name dedup uses `unordered_set` of FINAL
  emitted names plus a do-while suffix retry, so a literal
  `X_dup<n>` id cannot self-collide with a synthetic dedup name.

### 2. CDC: modport-width inference gap closed (US-R05)
Connection extractor now correctly forms an abs-path netKey on
both the consumer-side modport access (`bus.data` ->
`top.iface_inst.signal`) and the modport-expansion side, so
WidthChecker fires on the 8->16 truncation through interface
boundaries that previously slipped through silently. Verified on
the conn_modport_width_pos.sv fixture (now reports
`Zero-extension: 8 bits -> 16 bits`).

Implementation details:

* New `is_absolute` flag on `ResolvedExpr` bypasses the scopePath
  prefix when forming netMap_ keys.
* `resolveExpr(HierarchicalValue)` follows
  `ModportPortSymbol::internalSymbol->getHierarchicalPath()` for
  ModportPort symbols. Fallback path for null internalSymbol stays
  legacy-keyed (no false absolute claim).
* Modport-expansion side emits dual edges: legacy `Approximate`
  (scope-relative key) AND new `Direct` (abs-path key). Disjoint
  keyspaces -- no double-counting.

### 3. Connectivity: deterministic gen-array suffix-scan tie-break
The trailing `for (auto& [path, ff] : output_map)` last-resort
walk in `findFFByName` previously returned the first match in
unordered_map iteration order, which is implementation-defined.
Now collects all matches and picks the lexicographically-smallest
path via `std::min_element`. Same-FF deterministic across hash
function changes.

### 4. Test fixtures: designated initializers for FFNode
All positional `FFNode{"name", domain, nullptr, {}}` brace-inits
across test_edge_cases.cpp, test_coverage_gaps.cpp,
test_cdc_edge_cases.cpp, and test_connectivity_and_crossing.cpp
(22 sites total) converted to C++20 designated initializers.
Future field appends won't silently take defaults at any test
call site.

### 5. Atomic-commit hygiene
The commits between v0.2.8 and v0.3.0 each represent one
TDD red->green cycle or one bounded refactor. Independent
code-reviewer passes (architect + code-reviewer) approved the
work in two separate review windows, with all surfaced findings
closed before bump.

### 6. Test count
- 547/547 -> **548/548** ctest entries (added 1 collision_self case).

## Migration notes
- `ReportGenerator::generateSVA` signature changed from `void` to
  `[[nodiscard]] bool`. External users compiling against the
  public header should consume the return value.
- `ResolvedExpr` (private struct in ConnectionExtractor) gains
  `is_absolute`; downstream forks should mirror.
