# svlens v0.3.1 release notes

Patch release covering Round 34-35 review-driven hardening: 4-
perspective × 3-angle independent review (CDC, Lint, Structure,
Metric) surfaced 8 substantive findings, all closed.

## Highlights

### CDC analyzer
- **Self-shift TwoFF heuristic tightened**: requires `width >= 2`
  and `fanin.size() == 2` so 1-bit XOR-feedback storage and multi-
  source feedback no longer masquerade as a 2-FF synchronizer
  chain. Multi-bit shift register patterns (e.g. cdc_fifo_gray
  sync.sv) retain TwoFF classification.
- **Handshake naming heuristic extended**: Ac_cdc04 wide-bus check
  now recognizes AXI valid/ready, generic start/done, and FIFO
  enq/deq + push/pop alongside the existing gray/req/ack matches.
  Reduces false positives on modern industry RTL idioms.
- **Deep-hierarchy bailout surfaced**: `findFanin`'s depth >10 cap
  now emits a once-per-process stderr warning with the offending
  signal name instead of silently dropping resolution.
- **Parent-context chain bounded**: `g_parent_cont_chain` recursion
  capped at 32 levels with stderr warning on overflow. Pathological
  generators (deeply nested generate-for) no longer grow TLS state
  unboundedly.

### Metrics analyzer
- **Source location normalized**: `TransformExtractor::sourceLoc`
  now emits paths relative to the current working directory via
  `std::filesystem::relative`. Reports are portable across CI
  machines / users; `--baseline` diffs no longer drift on path
  changes alone.
- **always_latch / initial / final unsupported events named**:
  procedural-block kinds outside always_comb/always_ff are
  surfaced with the specific name and source location, instead of
  the previous generic "procedural_block" message.
- **LHS-concat double counting eliminated**: `assign {hi, lo} = expr`
  now produces one Alias node per element (with bit-range encoded
  in op_detail) instead of two (Slice + Alias). `raw_node_count`
  now matches the equivalent split-assign form.

### Code quality
- **`[[nodiscard]]` sweep**: applied to `AnalysisResult::*_count`,
  `validateCdcOptions`, `validateConnOptions`,
  `validateMetricsOptions`, `CompilationSession::hasCompilation`,
  `isPowerOf2`. Future callers that discard these returns now
  fail loudly at compile time.
- **SVA collision dedup hardened**: do-while retry loop bounded at
  `crossings.size() + 8`; on cap exhaustion, falls back to an
  address-based suffix for guaranteed unique property names.
  Existing `[collision]` test now pins the `_dup2` starting index
  to prevent silent drift.
- **Test fixtures: designated initializers**: 17 additional FFNode
  positional brace-inits across `test_cdc_edge_cases.cpp` and
  `test_connectivity_and_crossing.cpp` converted to designated
  initializers (Round 31 only covered 5 sites).

### Test count
- 547/547 -> **549/549** (+2 from new self_feedback +
  collision_self regression cases).

## Migration notes
No public API changes vs. v0.3.0. The `[[nodiscard]]` sweep is
forward-compatible (current callers all consume the returns).
