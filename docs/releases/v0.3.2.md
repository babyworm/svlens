# svlens v0.3.2 release notes

Patch release adding 6 lowRISC style-guide checks across three
slang representation layers (AST / Syntax tree / SourceText).
The new ConventionChecker now covers the full lowRISC SystemVerilog
Coding Style guide via paired GOOD/BAD fixtures.

## Highlights

### lowRISC style coverage — 3-layer scanner

* **Reset polarity** (US-39A, AST): always_ff using
  `@(posedge clk, negedge rst_n)` (comma) or active-high reset is
  flagged. Canonical `or` form remains clean.
* **FF d-suffix** (US-39B, AST): combinational input feeding a
  `_q` register should be named `_d`. Per-module pair detection
  across always_ff and always_comb / continuous assigns.
* **`.*` named-port instantiation** (US-39C, Syntax): wildcard
  port connections are flagged. AST loses this distinction since
  slang expands `.*` into named connections during elaboration;
  the new StyleSyntaxScanner walks the raw syntax tree.
* **Bare integer literal width** (US-39D, Syntax): `assign x = 2`
  flagged; `8'd2` and `'0` clean. Skips loop indices and
  parameter defaults to avoid false positives.
* **Source-text rules** (US-39E, raw text): line length (default
  off; opt-in via `max_line_length`), hard tabs, trailing
  whitespace.
* **File-naming rules** (US-39F, raw text): one module per file +
  file basename matches primary module name. Both opt-in.

### Architecture
* New `src/StyleSyntaxScanner.{h,cpp}` runs after the AST walk.
  Per-top-module BFS over HierarchyInstantiationSyntax scopes the
  scan so sibling modules in shared fixture files don't
  cross-contaminate.
* New `src/SourceTextScanner.{h,cpp}` performs line-by-line
  scanning of input files. CR-LF aware; configurable limits via
  the existing `ConventionRules` YAML.
* `StyleObservation::Kind` enum grows to 14 values covering all
  rounds. Stable enum tagging for downstream consumers.

### YAML schema additions (all opt-in, defaults disabled)
* `max_line_length` (int; 0 disables)
* `prohibit_hard_tabs` (bool)
* `prohibit_trailing_whitespace` (bool)
* `prohibit_multiple_modules_per_file` (bool)
* `enforce_file_module_match` (bool)

### Test count
551 ctest (was 550) — added the [conn][runner][source_text] case
covering the new scanners with a synthesized strict YAML.

## Migration notes
No public API changes vs. v0.3.1. The new scanners are wired
internally to ConnRunner; existing yaml configs continue to work
unchanged because all new rules default to disabled.

## Known follow-ups
* Architect-recommended: promote `SourceTextScanner` +
  `StyleObservation` to `src/style/` shared module so CDC and
  lint modes can reuse the source-text checks. Defer until a
  concrete CDC/lint user story requests it.
* `StyleObservation::location` is a default `SourceLocation` for
  source-text-only kinds (line number lives in `detail` string).
  Adding an explicit `lineNumber` field would let downstream
  renderers (DOT, JSON) print precise locations.
