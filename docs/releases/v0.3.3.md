# svlens v0.3.3 release notes

Patch release that hardens the v0.3.2 lowRISC style guide implementation
through two structured review cycles and one CI-fix pass. No new
checks land; every change is correctness, robustness, observability,
or de-duplication of code introduced in v0.3.2.

The release also unblocks the project's CI pipeline, which had been
red since several commits before v0.3.2 due to two latent build issues
that pre-existing local development environments were masking.

## Highlights

### CI restoration

* `svlens-cdc-lib` now links `${YAML_CPP_TARGET}` explicitly. Previously
  it relied on a system-installed `yaml-cpp` exposing the header through
  a transitive include path, which broke the `build-and-test` job on
  CI's FetchContent-only environment.
* `slang::slang`'s `INTERFACE_COMPILE_DEFINITIONS` exports
  `$<$<CONFIG:Debug>:SLANG_DEBUG>`, which assumes a Debug consumer
  matches a Debug-built slang. `setup-deps.sh` always builds slang in
  Release, so a Debug build of svlens (the coverage CI job) ended up
  with mismatched class layouts in `IntervalMap`, `SourceLocation`, and
  `CommandLine` — surfacing as a runtime
  `std::runtime_error{"No options defined"}` from `Driver`. The
  generator-expression entry is now stripped from slang's interface
  property; consumers see a consistent ABI regardless of
  `CMAKE_BUILD_TYPE`. The strip is regex-based so any future
  `$<$<CONFIG:Debug>:...>` macro is also caught. Set
  `SVLENS_KEEP_SLANG_DEBUG=ON` to opt out if your slang was built
  Debug.
* CMake emits a STATUS message when the strip activates, so
  contributors notice the workaround instead of grep-debugging the
  symptom.

### Detector correctness

* **Bare integer literal context filter**:
  - Skips `ParameterValueAssignment` and `ParameterPortListSyntax`
    so `.WIDTH(8)` overrides aren't flagged.
  - Skips ancestor `ConcatenationExpression` and replication contexts
    where width is already constrained.
  - Exempts the literals `0` and `1` (lowRISC §6.7).
  - Ancestor walk depth raised from 20 to 64 to avoid silent false
    negatives in deeply nested ternaries / generate bodies.
* **Active-high reset detection**:
  - No longer hardcodes the `rst_p` substring. Drives off
    `ConventionRules::resetPattern` so signals like `rst_pulse`,
    `rst_pin`, and bare `rst` no longer false-positive / -negative.
  - Strips trailing `[...]` index before suffix testing so
    `rst_n[i]` / `rst_ni[0]` classify as active-low resets.
* **`_q` suffix detector** now requires the `_q` token at end-of-name
  (or `_q[0-9]+$` digit tail). Names like `data_qual_next` are no
  longer flagged just because they contain the substring `_q`.
* **Source-text scope**: `SourceTextScanner` now respects the
  `--top` argument. Files containing zero modules reachable from the
  requested top are skipped entirely. Per-line rules remain file-
  scoped (a single line could span module declarations) — see the
  doc-comment in `src/SourceTextScanner.h`.
* **Tab-end deduplication**: A line ending in `\t` now emits only
  `HardTab`, not duplicate `HardTab + TrailingWhitespace`.
* **Hierarchical reads** to `_q` registers (struct-field writes
  with `.` in the base) no longer trigger `MissingQSuffix`.

### YAML hardening

The convention YAML loader is now resistant to malformed user input:

* `YAML::LoadFile` is wrapped in `try/catch`. Adversarial files with
  cyclic anchors / EOF mid-document return a clean `runtime_error`
  instead of aborting the process.
* Per-field `tryAssign<T>` helper extracts each scalar with its own
  catch. A typo in `max_line_length: nope` no longer disables later
  valid keys (`prohibit_hard_tabs: true`) depending on key order.
* `getString` wraps `.as<std::string>()` per field.
* Nested `input` / `output` / `instance` `prefix` accesses are
  guarded by parent-shape (`IsMap`) and child-shape (`IsScalar`)
  checks before conversion. A malformed `input: scalar_not_a_map`
  no longer throws past the loader. Triplicated guard collapsed
  into a single `tryAssignNestedPrefix` helper.
* `runConnWithCompilation` wraps the `loadConventionRules` +
  `SourceTextScanner::scan` call in `try/catch`. A scan-time exception
  degrades to a stderr warning instead of crashing conn mode before
  report generation.

### Observability

`StyleObservation`, `Issue`, and `JsonReport` now carry structured
`lineNumber` / `columnNumber` fields end-to-end. Source-text rules
populate them from the scanner's line tracker; AST rules use the
global `Compilation::getSourceManager()` (not per-tree, which
previously returned 0 for `\`include`d headers). JSON consumers get
`"line": 13, "column": 8` instead of regex-parsing the human detail
string.

`StyleObservation::Kind` enum gains four dedicated values:
`MissingQSuffix`, `BannedStateType`, `MissingCaseDefault`,
`MissingUniqueCase`. Previously these distinct rules collapsed onto
`LegacyAlwaysBlock`, defeating downstream filtering.

### Defensive hardening

* `ConventionChecker` regex patterns now live in a per-instance
  `std::optional<std::regex>` cache. A pattern compiles once, not on
  every `check()` call.
* User-supplied regex patterns longer than 256 chars or matching a
  catastrophic-backtracking heuristic (`(...|...[+*])[+*]`,
  3+ consecutive quantifiers) are rejected with an INFO. Defensive
  measure against ReDoS in adversarial `convention.yaml`.
* Line-length comparison uses `size_t` (no sign-flip on >2 GiB lines).
* The ReDoS heuristic regex itself is constructed once at namespace
  scope (was previously re-built inside the per-call lambda).

### Architecture cleanup

* `collectReachableModules` BFS — previously duplicated in
  `SourceTextScanner` and `StyleSyntaxScanner` — is now shared via
  `src/StyleReachability.{h,cpp}`. Future scope drift between the
  two scanners is now a one-edit problem rather than a two-edit
  silent-divergence problem.
* The legacy no-`topModule` `SourceTextScanner::scan` overload had
  no callers in `src/` and has been deleted. Keeps `--top`
  reachability gating mandatory and eliminates a footgun for
  filelist-with-vendor-IP runs.
* Both scanners use `compilation.getSourceManager()` (the global
  source manager) and `getRawFileName(buffer)` for path rendering,
  so a single observation no longer has mismatched `scopePath`
  values when the same line is reported by both scanners.

## YAML schema additions

No new keys vs. v0.3.2. All v0.3.2 keys remain stable.
The shipped example `examples/styles/lowrisc.yaml` now wires the opt-in
source-text and file-naming keys (`max_line_length`, `prohibit_hard_tabs`,
`prohibit_trailing_whitespace`, `prohibit_multiple_modules_per_file`,
`enforce_file_module_match`) so users copying the example get
out-of-the-box enforcement instead of silently-disabled rules.

## Test count

`make test` reports **560/560 passing** — up from 553 at the v0.3.2
push. New cases cover:

* Source-text reachability: clean fixture emits zero observations,
  unrelated-file reachability is suppressed, in-file sibling
  reachability is preserved (file-level scope locked-in).
* `FileNameMismatch` is now asserted explicitly; previously the rule
  was emitted but no test referenced it.
* Multiple malformed-YAML edge cases: bad scalar, bad string field,
  bad nested prefix value, bad nested parent shape, bad-scalar-
  doesn't-skip-later-keys.
* JSON `"line": <n>` / `"column": <n>` substring assertions for
  source-text observations.
* New paired `q_suffix_anchor_check` and
  `reset_polarity_bracketed_good` fixtures lock in the corrected
  detector behaviors.

## Migration notes

No public API changes vs. v0.3.2. JSON output gains optional `line`
and `column` keys on issues that have a known location; consumers
that don't read those keys are unaffected. The legacy 3-arg
`SourceTextScanner::scan(compilation, rules, graph)` overload is
removed; the only external caller in svlens already used the
4-arg form.

## Known follow-ups

* Promote `SourceTextScanner` and `StyleObservation` to a `src/style/`
  shared module so `cdc` and `metrics` modes can reuse the
  source-text / convention checks. Currently bound to `conn`.
* Replace the `looksLikeRedos` pattern-shape heuristic with a
  runtime-budget guard or RE2. The current heuristic is bypassable
  by classic catastrophic patterns (`(a+)+`, `(?:a+){10,}`, etc.)
  that don't match the 3-quantifier-run signature.
* Bump `yaml-cpp` past 0.8.0 once a tagged release with the post-
  CVE-2024-35325/35326 fixes ships.
* Extract the AST-side style observation collection out of
  `ConnectionExtractor` into a peer pass; the extractor currently
  conflates connectivity extraction with style-smell collection.
