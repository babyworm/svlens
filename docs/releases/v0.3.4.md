# svlens v0.3.4 release notes

Patch release that extends the v0.3.3 lowRISC style guide implementation
through targeted regression hardening and reproducible-benchmark
scaffolding. No new check kinds land; every change is correctness,
robustness, fixture pinning, scanner observability, infrastructure, or
end-to-end reproduction tooling.

## Highlights

### YAML loader regression coverage

The v0.3.3 try/catch hardening was previously only exercised by happy-
path tests. v0.3.4 locks in non-crash behavior on adversarial input:

* Self-referencing / cyclic anchors return defaults or surface
  `YAML::Exception` via the existing try/catch — no abort.
* Deeply nested anchors (~50 levels) — no stack overflow.
* Empty `{}` body — `ConventionRules` default-initializes cleanly.
* Mixed unknown + valid keys — unknown silently ignored, valid keys
  still apply.
* Malformed scalar where a map is expected, malformed nested-prefix
  shape, and a bad-scalar that must not skip later valid keys are now
  asserted explicitly.

### Encoding edge-case fixtures

`SourceTextScanner` now has dedicated fixtures and assertions for:

* BOM-prefixed source files
* CR-LF line endings (the `no_eol` fixture is reused with pinned
  long-line and CR-LF counts so future scanner changes that silently
  re-tokenize line endings surface as count drift)
* Files missing a trailing newline (`no_eol_top`)
* Long-line boundary conditions

### Column propagation for source-text observations

`docs/schema/connect_report.md` advertised optional `column` for
source-text issues since v0.3.3, but the scanner only emitted
`lineNumber` so JSON consumers silently lost it. v0.3.4 tracks per-rule
offsets end-to-end:

* `HardTab`: column of the first tab on the line.
* `TrailingWhitespace`: column where trailing whitespace begins.
* `LineTooLong`: `maxLineLength + 1` (first violating column).
* `MultipleModulesPerFile` / `FileNameMismatch`: column 1 (line-anchored).

A lock-in test asserts a known column in JSON output for the existing
tab-violation fixture.

### Cross-scanner consistency pinning

A new test asserts that `StyleSyntaxScanner` and `SourceTextScanner`
emit identical `scopePath` strings for issues in the same file —
catching regressions to the pre-v0.3.3 `getFileName` /
`getRawFileName` split. Three high-value fixtures (`ports_bad`,
`width_lit_bad`, `case_bad`) move from `>= 1` substring presence to
exact violation counts so silent emit drift surfaces as a test
failure instead of noise.

### Example yaml reach-through

`examples/styles/lowrisc.yaml` now ships with the v0.3.2 source-text /
file-naming opt-ins (`max_line_length`, `prohibit_hard_tabs`,
`prohibit_trailing_whitespace`, `prohibit_multiple_modules_per_file`,
`enforce_file_module_match`) wired to lowRISC defaults. A new
regression test loads the example file directly and verifies it trips
on `lowrisc_source_text_violations.sv`, so contributors copying the
example get out-of-the-box enforcement.

Fixture comments around `_q` semantics were rewritten to separate
"mid-string match" and `MissingQSuffix` — two distinct rules that the
previous comments conflated. The example also clarifies that
`instance_prefix` accepts only a single value (no list form) so users
do not pass `"u_,inst_"` expecting alternation.

### Parallel-ctest isolation

Test scripts that previously shared `${TMPDIR}/svlens-…` paths across
runs now use per-PID + per-counter temp directories, removing a race
where `ctest -j` could clobber sibling outputs.

### OpenTitan benchmark scaffold

A reproducible end-to-end benchmark target now lives under
`bench/opentitan/`:

* `fetch.sh` clones OpenTitan at a pinned tag
  (`earlgrey_silver_release_v5`).
* `gen_filelist.py` walks FuseSoC `.core` files (handling the non-YAML
  `CAPI=2:` header line and full-VLNV vs. version-stripped dependency
  lookup) to emit per-target `.f` filelists.
* `run.sh` invokes svlens against each generated filelist.
* `evaluate.py` computes recall / precision against curated
  `golden/*.yaml` for the `aes`, `hmac`, `uart`, and `top_earlgrey`
  cones.
* `make bench` ties the four steps together.

This is developer-facing tooling; no svlens runtime behavior changes
from it. `docs/superpowers/specs/2026-04-03-opentitan-benchmark-design.md`
documents the design and the `golden/` acceptance contract.

### CI: Node.js 24 migration

Hosted GitHub Actions runners drop Node 20 support entirely on
2026-09-16 and force every Node 20 action to Node 24 on 2026-06-02.
v0.3.4 lands ahead of the cutover so neither the warning annotations
nor the eventual hard failure block CI:

* `actions/upload-artifact` bumped from v4 (Node 20) to v6 (Node 24
  default).
* `codecov/codecov-action` bumped from v5 to v6 — also resolves the
  `actions/github-script@60a0d83` (Node 20) deprecation that the v5
  codecov-action pinned internally.

### Workspace meta

`CLAUDE.md` now references `AGENTS.md` so OMC / OMX coding harnesses
auto-load the workspace operating contract. No effect on svlens
runtime.

## YAML schema additions

No new keys vs. v0.3.3. The shipped example
`examples/styles/lowrisc.yaml` finally exposes the v0.3.2 source-text
and file-naming keys that had been silently default-disabled.

## Test count

`make test` passes. v0.3.4 adds adversarial-YAML, encoding-edge,
exact-count, scopePath-consistency, and column-emit assertions on top
of the v0.3.3 baseline. The exact count is reported by CI's
`build-and-test` job rather than pinned in this document — several of
the new shell-test paths exercise CI-only fixtures, so a locally-pinned
number would drift on developer machines.

## Migration notes

No public API changes vs. v0.3.3. JSON output now populates `column`
on source-text issues that already emitted `line` — consumers that
ignored the field continue to work; consumers that did regex-parse it
out of the human detail string can switch to the structured field.

CI consumers running self-hosted Actions runners pinned below Actions
Runner `2.327.1` must upgrade before adopting this release into their
own pipelines: `actions/upload-artifact@v6` requires that runner
version. GitHub-hosted runners already meet the bar.

## Known follow-ups

Carry-overs from v0.3.3:

* Promote `SourceTextScanner` and `StyleObservation` to a `src/style/`
  shared module so `cdc` and `metrics` modes can reuse the source-text
  / convention checks. Currently bound to `conn`.
* Replace the `looksLikeRedos` pattern-shape heuristic with a
  runtime-budget guard or RE2. The current heuristic is bypassable by
  classic catastrophic patterns (`(a+)+`, `(?:a+){10,}`) that don't
  match the 3-quantifier-run signature.
* Bump `yaml-cpp` past 0.8.0 once a tagged release with the
  post-CVE-2024-35325 / 35326 fixes ships.
* Extract the AST-side style observation collection out of
  `ConnectionExtractor` into a peer pass.

New in v0.3.4:

* Wire `make bench` into CI as a non-blocking job once OpenTitan-clone
  bandwidth and runner caching policy are agreed.
* Promote `examples/styles/lowrisc.yaml` end-to-end tests into a
  format-stable golden-comparison so a single new check landing
  doesn't require touching the example.

## Recommended validation

```bash
make test
ctest --test-dir build --output-on-failure
```

Optional, requires network and ~2 GiB scratch:

```bash
make bench
```
